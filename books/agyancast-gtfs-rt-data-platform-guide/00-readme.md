この本は、`agyancast` という実プロジェクトを題材に、**GTFS-RTを「システムとして動かす」ために必要な流れ**を順番に理解する本です。

- どんな問題意識で作り始めたか（何を可視化したいか）
- GTFS / GTFS-RT をどう読むか（今回どのフォーマットのどの項目を使うか）
- GTFS-RTをシステムに落とし込む全体フロー（収集→保存→変換→配信）
- AWS上でどう蓄えると後で効くか（再処理・デバッグ・配信・分析に耐えるデータの置き方）

BIN（protobuf）の中身や、JSON（JSONL: 1行1JSON）へ落とす具体の実装は、まずこの全体フローを押さえた後に後半で扱います。

## 先に結論

このプロジェクトで一番重要だった判断は、次の3つです。

1. 変換をやり直せるように、取得したBINをそのまま失われない形で保管する（変換前の元データ）
2. 集計とデバッグがしやすいように、必要項目を抜き出してイベントログとして蓄積する（1観測=1レコード）
3. 使い方が違うので、画面向けのスナップショットと分析向けの履歴・集計を分けて出力する（最新 vs 時系列）

つまり、「いきなり最適解を目指す」のではなく「壊れにくい土台から積む」方針です。

読み終えた時に、次を説明できれば成功です。
