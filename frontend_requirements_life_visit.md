# フロント追加要件整理（生活/来訪タブ）

更新日: 2026-02-17

## 1. 目的
- フッターで **生活** と **来訪** を切り替えられる2タブ構成にする。
- 生活: 既存の「おすすめモール + 混雑度」体験を継続。
- 来訪: 県外来訪者向けに「バス遅延情報（まずは阿蘇くまもと空港リムジンバス）」を提供。

## 2. UI要件

### 2.1 フッタータブ
- 固定フッターに2タブ:
  - `生活`
  - `来訪`
- タップでページ全体を切り替える（アンカー遷移ではなくビュー切替）。

### 2.2 生活タブ
- 既存機能を維持:
  - おすすめ表示
  - モール一覧（混雑/遅延）
  - 日次推移グラフ
  - 補助地図

### 2.3 来訪タブ
- 地図は表示しない。
- 表示要素:
  1. 阿蘇くまもと空港リムジンバスの現在遅延（最優先）
  2. 将来遅延（1時間後/3時間後、当面は準備中表示可）
  3. 当日遅延推移（データがあれば表示）

## 3. データ契約（来訪タブ）

### 3.1 現在値（新規）
- `GET /data/visitor_airport_latest.json`

想定形式:
```json
{
  "updated_at": "2026-02-17T08:30:00+09:00",
  "route_id": "aso_airport_limousine",
  "route_name": "阿蘇くまもと空港リムジンバス",
  "status": "on_time",
  "delay_sec": 240,
  "note": "道路混雑により最大5分程度の遅れ",
  "predictions": {
    "h1_sec": null,
    "h3_sec": null
  }
}
```

`status` 語彙:
- `on_time`
- `slight_delay`
- `delayed`
- `suspended`
- `unknown`

### 3.2 当日推移（新規・任意）
- `GET /data/visitor_airport_daily.json`

想定形式:
```json
{
  "date": "2026-02-17",
  "timezone": "Asia/Tokyo",
  "hours": ["00","01","...","23"],
  "delay_min": [null, null, 2.0, ...]
}
```

## 4. 地図位置ずれ対応
- マップピン座標に補正レイヤを追加し、表示ずれが出てもフロント側で微調整可能にする。
- 地図背景の描画スケールを明示し、オーバーレイとの1:1対応を固定する。

## 5. 完了条件
- フッターに `生活 / 来訪` の2タブが表示される。
- 来訪タブで空港リムジンバスの現在遅延が表示される。
- 来訪タブで将来遅延（少なくとも準備中文言）が表示される。
- 生活タブの既存機能が継続して動作する。
- 地図のピン位置が従来より安定する（補正可能な実装になっている）。
